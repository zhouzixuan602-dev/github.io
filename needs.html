Post a Need

“Needs” are small, concrete collaboration requests. Write them clearly and the network can match you faster.

What belongs here

  • Technical help (analysis, modeling, hardware, software)
  • Access to data, equipment, participants, or labs
  • Short-term research or co-development tasks
  • Funding, validation, or deployment gaps

Where needs live (current MVP)

For now, all needs are stored in a single open document: needs.md. This keeps everything transparent, versioned, and auditable.

You can open the file, add a new section, and commit.

Posting checklist (MVP)

Treat a need like an executable task, not a vague idea.

  1. One-line title
    Goal + context
  2. Inputs / Outputs / Acceptance
    Provide → Need → Done means…
  3. Time window
    Availability, urgency, async vs sync
  4. What you offer
    Authorship, funding, data, compute, credit
  5. Next step
    Email, GitHub issue, short call

Flow: draft → add to needs.md → commit → visible
